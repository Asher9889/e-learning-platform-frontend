# Product

## Register

product

## Users

University administrators, faculty, and students — plus organizations purchasing a controlled LMS for internal training. Context: institutional environments where credibility, data integrity, and structured workflows are non-negotiable. Primary jobs: course/content management, live class scheduling, enrollment tracking, assessment delivery, progress reporting.

## Product Purpose

A controlled Learning Management System that institutions trust for delivering structured education at scale. Success means: admins configure programs without friction, teachers run live sessions reliably, students navigate coursework without confusion, and the platform feels like infrastructure — not a consumer app.

## Brand Personality

Authoritative, Structured, Academic. Voice: precise, calm, confident. Tone: respectful of expertise, never patronizing. Emotional goals: trust, clarity, competence.

## Anti-references

Legacy LMS (Moodle, Blackboard classic) — dated visual language, cluttered information density, opaque navigation, dense tables without hierarchy, confusing multi-step workflows. Also avoid: consumer ed-tech playfulness (gamification, bright gradients, cartoon illustrations), generic SaaS blandness (cream backgrounds, hero-metric cards, indistinguishable dashboards), over-designed marketing aesthetics (glassmorphism, animated blobs, gradient text).

## Design Principles

1. **Clarity over density** — Information architecture wins. Every screen has a clear primary action; secondary actions recede. No mystery meat navigation.
2. **Institutional trust signals** — Visual language conveys stability and rigor: restrained color, purposeful typography, consistent spacing, predictable interactions.
3. **Expert efficiency** — Power users (admins, teachers) need keyboard shortcuts, bulk actions, dense-but-scannable tables, and workflows that don't force clicks.
4. **Structured flexibility** — The system accommodates varied course structures (semesters, modules, cohorts) without becoming form-heavy configuration.
5. **Accessible by default** — WCAG 2.1 AA is the baseline, not a checklist. Color alone never carries meaning; focus states are visible; motion is reducible.

## Accessibility & Inclusion

WCAG 2.1 AA compliance. Color contrast ≥4.5:1 for body text, ≥3:1 for large text. Visible focus indicators on all interactive elements. Semantic HTML structure. Reduced-motion alternatives for all animations. Screen reader tested navigation and live regions for dynamic updates.
